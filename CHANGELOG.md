# Changelog
All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html),
and is generated by [Changie](https://github.com/miniscruff/changie).


## 0.3.3 - 2026-04-18
### Fixed
* tui: keep guide marker inside the filled region when utilization exceeds the guide on narrow bars (#60)

## 0.3.2 - 2026-04-17
### Added
* New `.devcontainer/` (Go 1.25) and `scripts/e2e-smoke.sh` — catches CLI regressions unit tests miss
### Fixed
* self-update: preserve the executable bit on the installed binary — previously non-executable after update

## 0.3.1 - 2026-04-17
### Fixed
* self-update and `scripts/install.sh`: point release lookup at `rudolfjs/agent-quota` (was `schnetlerr/agent-quota`)

## 0.3.0 - 2026-04-17
### Added
* New `agent-quota self-update` (aka `aq self-update`) — atomically updates to the latest release with sha256 verification, supports `--check`/`--force`/`--pre` (Linux/amd64 only)
### Fixed
* Claude provider: normalize `extra_usage` cents→dollars for sub-cent per-token billing and non-USD accounts (AUD)

## 0.2.2 - 2026-04-11
### Changed
* Gemini provider: delegate OAuth token refresh to the gemini CLI instead of using hardcoded client credentials, aligning with the Claude provider pattern

## 0.2.1 - 2026-04-10
### Fixed
* Release workflow no longer re-triggers itself when the app token force-pushes the tag

## 0.2.0 - 2026-04-10
### Changed
* Release pipeline auto-batches changie fragments and pushes changelog to main via GitHub App token
* Branch protection migrated from classic rules to GitHub rulesets with app bypass

## 0.1.1 - 2026-04-10
### Added
* New `@jules-issue` workflow triages issues — gives maintainers a structured pre-implementation assessment (#23)
* Adds TUI budget guide markers and a persisted setting to hide them
* New `aq` short alias — `scripts/install.sh` and `make local-install` now symlink `aq` → `agent-quota` (#9)
### Changed
* Update README.md with Claude re-authentication instructions for 429 and 403 errors
* release: automate changie batch/merge in GitHub Actions — just push a tag to release
### Fixed
* Replace undefined `die` with `fail` in install.sh TTY fallback
* Fix incomplete file permission check for credentials
* limit supported release installs to linux/amd64
* Fix CI changie job shallow-fetch causing 'no merge base' errors
* Include HTTP status code in API error messages and allow Ctrl+R to override retry backoff timers
* claude: cache HTTP 429 Retry-After and surface rate limits in error output
* claude: add manual backoff reset for ctrl+r and --force, and surface safe headless error details
* claude: prevent --force and ctrl+r from being defeated by re-persisted backoff, cap Retry-After at 5 minutes

## 0.1.0 - 2026-03-30
### Added
* New `changie` workflow manages unreleased fragments and release-note generation
* New GitHub CI runs `lefthook`, lint, tests, and builds for pull requests
* New GitHub Releases build cross-platform binaries and ship a `scripts/install.sh` installer
* `agent-quota` dashboard auto-refreshes on a configurable interval
* `agent-quota` dashboard scrolls through provider cards with a visible scrollbar
* New `make` targets simplify local build, test, and release checks
* New persistent dashboard settings save shown providers, card order, header visibility, and refresh interval
* New `copilot` provider shows GitHub Copilot usage and premium request quotas
### Changed
* Dashboard now supports manual refresh with `Ctrl+R` and a 5-minute refresh option
* GitHub Releases now verify tags point to `main` before publishing binaries
